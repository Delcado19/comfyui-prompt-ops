# Architecture

ComfyUI Prompt Ops uses **Espanso** as a modular prompt composition engine.

The system allows prompts to be built from reusable components instead of writing full prompts manually.

---

# System Overview

```
User typing
   │
   ▼
Espanso Trigger
   │
   ├─ Direct Snippet Expansion
   │
   └─ Prompt Builder (:prompt)
            │
            ▼
       Generated Prompt
            │
            ▼
          ComfyUI
```

The system is composed of three main layers:

1. **Snippet Library**
2. **Prompt Builder**
3. **Developer Toolchain**

---

# Snippet Library

The prompt components are stored in:

```
snippets/
```

Each YAML file represents a category of prompt elements.

Example:

```
snippets/
  comfy_context.yml
  comfy_characters.yml
  comfy_scene.yml
  comfy_camera.yml
  comfy_lighting.yml
  comfy_style.yml
  comfy_quality.yml
  comfy_negative.yml
  comfy_nsfw.yml
```

Each file contains multiple Espanso snippet definitions.

Example snippet:

```yaml
matches:
  - trigger: ":ctx_portrait"
    word: true
    replace: "portrait, centered composition, subject facing camera"
```

---

# Prompt Builder

The prompt builder allows users to construct prompts interactively.

Trigger:

```
:prompt
```

The builder presents multiple selection fields and generates a final prompt from them.

The builder configuration is stored in:

```
snippets/zz_prompt_builder.yml
```

This file is automatically generated by:

```
scripts/generate_prompt_builder.ps1
```

---

# Developer Toolchain

Development scripts are located in:

```
scripts/
```

These scripts provide tooling for:

- YAML validation
- snippet validation
- duplicate trigger detection
- snippet documentation generation
- prompt builder generation

---

# Espanso Integration

Snippets become active only after being copied to the Espanso match directory.

Default path:

```
%APPDATA%\espanso\match
```

Example:

```
C:\Users\<USER>\AppData\Roaming\espanso\match
```

Snippets can be installed using:

```powershell
.\scripts\install_snippets.ps1
```

---

# Development Pipeline

The development pipeline is executed with:

```powershell
.\scripts\dev.ps1
```

The pipeline performs:

1. YAML syntax validation
2. snippet validation
3. duplicate trigger detection
4. snippet documentation generation
5. prompt builder generation

This ensures the snippet system remains consistent and maintainable.
